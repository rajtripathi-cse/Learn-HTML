üìÅ File Paths in Web Development

  • Websites are collections of files (HTML, CSS, JS, images, etc.) organized in folders.
  • Relative Paths: Used to link files within your own site.

    • Same folder: about.html
    • Sub-folder: images/logo.png
    • Parent folder: ../images/logo.png

  • Absolute Paths: Used for external resources (e.g., https://example.com/image.jpg).
  • Security Note: Browsers restrict access to local file systems to prevent malicious attacks.

HTML Boilerplate

    
<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Document</title>
  </head>
  <body></body>
</html>
  • Each tag serves a specific purpose for rendering, accessibility, SEO, and mobile responsiveness.
  • Browsers are forgiving, but skipping boilerplate leads to unpredictable behavior (quirks mode, broken characters, poor mobile experience).

    • id: Unique identifier (used once per page).
    • class: Reusable label for styling multiple elements.

  • span tag: Inline container for styling parts of text without breaking layout.
