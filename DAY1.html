How Does Internet Work

Principle 1: The Need to Share Information

The most fundamental truth is that we have information on one computer (A) and we want it to be on another computer (B). At first, we can transfer with USB drives but that doesn’t scale. The logical solution is direct connections with cables that transmit signals (1s and 0s). This creates the most basic form of a network.

Principle 2: The Problem of Scale

Direct links between every computer don’t scale. The solution: a central device (Switch). Each computer connects to the switch, enabling Local Networks.

Principle 3: The Problem of Connecting Networks

Local Networks can’t scale globally. The solution: Routers connect multiple networks together, forming a “network of networks” — the Internet.

Principle 4: The Problem of Finding Anyone (Addressing)

We need unique addresses to find computers. Solution: IP Address. Routers forward messages based on IPs, ensuring packets reach the destination.

Principle 5: The Problem of Reliability and Fairness

Large files are split into smaller Packets with sender, receiver, and sequence number. They travel independently and reassemble at destination. Protocol used: TCP/IP.

Summary from First Principles

  • Share data → Direct connection
  • Connect many computers → Local Network
  • Connect many networks → Routers (Internet)
  • Find a computer → IP Address
  • Reliable transfer → TCP/IP and Packets

IP Address

IPv4: 32-bit, ~4.3 billion addresses (e.g., 172.217.16.142). Problem: exhaustion.

Public IP: Globally unique, assigned by ISP.

Private IP: Used inside local networks (e.g., 192.168.x.x).

IPv6: 128-bit, ~340 undecillion addresses (example: 2001:0db8:85a3::8a2e:0370:7334).

MAC Address

Unique 48-bit hardware identifier (example: 3C:22:FB:A3:B4:C5).

  • Windows: ipconfig /all or getmac /v
  • MacOS: ifconfig

Port Numbers

Range: 0–65535. Categories:

  • Well-Known Ports (0–1023): HTTP 80, HTTPS 443, FTP 21, SSH 22, DNS 53
  • Registered Ports (1024–49151): React/Node.js 3000, MySQL 3306, PostgreSQL 5432, MongoDB 27017
  • Dynamic/Ephemeral (49152–65535): Used temporarily by clients

DNS (Domain Name System)

Translates domain names into IPs.

  1. Check Local Cache (browser, OS, router)
  2. Recursive Resolver (ISP or Google’s 8.8.8.8)
  3. Hierarchical Search: Root → TLD → Authoritative Server
  4. Journey Back: Resolver caches and sends IP to browser
  5. Final Connection: Browser connects to server using IP + Port
